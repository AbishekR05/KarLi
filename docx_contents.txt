API Specification Document
Jewellery Ornaments E-Commerce Website
Version 1.0  |  March 2026
Base URL: http://localhost:5000/api
1. Overview
This document defines every REST API endpoint exposed by the KarLi Fashion backend. The React frontend communicates exclusively through these endpoints. All requests and responses use JSON. The base URL for all endpoints is http://localhost:5000/api in development and the live server URL in production.
Base URL (dev)
http://localhost:5000/api
Data Format
JSON — all requests and responses
Auth Method
Bearer token in Authorization header
Token Storage
localStorage — 'karli_user_token' / 'karli_admin_token'
User Token Expiry
7 days
Admin Token Expiry
24 hours
File Uploads
multipart/form-data for product image endpoints
Error Format
{ success: false, message: 'description of error' }
Success Format
{ success: true, data: { ... } }
1.1 Auth Levels
Public
No token required — anyone can call this endpoint
User
Requires valid karli_user_token in Authorization header
Admin
Requires valid karli_admin_token and role === 'admin'
1.2 Standard Error Responses
400 Bad Request
Missing or invalid fields in the request body
401 Unauthorized
No token provided or token is expired / invalid
403 Forbidden
Valid token but insufficient role (e.g. customer hitting admin route)
404 Not Found
The requested resource does not exist
500 Server Error
Unexpected server-side error

1. Overview
This document defines the complete database schema for KarLi Fashion. It covers all collections, fields, data types, constraints, and relationships. The schema is written in Prisma Schema Language (PSL) and targets MongoDB Atlas as the cloud database.
ORM
Prisma v5.x
Database
MongoDB Atlas (free cloud tier — works in both dev and prod)
Schema Apply
npx prisma db push (no migration files with MongoDB)
Schema File
server/prisma/schema.prisma
Apply Schema
npx prisma db push (no migration files with MongoDB)

3. Prisma Schema File
Save this as server/prisma/schema.prisma exactly as written below.
generator client {
  provider = "prisma-client-js"
}
datasource db {
  provider = "mongodb"
  url      = env("DATABASE_URL")
}
// ─── USER ───────────────────────────────────────────────
model User {
  id           String    @id @default(auto()) @map("_id") @db.ObjectId
  fullName     String
  email        String    @unique
  passwordHash String
  phone        String?
  role         Role      @default(CUSTOMER)
  status       UserStatus @default(ACTIVE)
  createdAt    DateTime  @default(now())
  updatedAt    DateTime  @updatedAt
  orders    Order[]
  wishlist  Wishlist[]
}
// ─── PRODUCT ────────────────────────────────────────────
model Product {
  id               String    @id @default(auto()) @map("_id") @db.ObjectId
  name             String
  description      String
  category         Category
  material         String
  weightGrams      Float?
  price            Float
  stockQuantity    Int       @default(0)
  images           String    // JSON array of image paths
  status           ProductStatus @default(ACTIVE)
  discountEnabled  Boolean   @default(false)
  discountType     DiscountType?
  discountValue    Float?
  discountStart    DateTime?
  discountEnd      DateTime?
  createdAt        DateTime  @default(now())
  updatedAt        DateTime  @updatedAt
  orderItems  OrderItem[]
  wishlist    Wishlist[]
}
// ─── ORDER ──────────────────────────────────────────────
model Order {
  id              String    @id @default(auto()) @map("_id") @db.ObjectId
  userId          String    @db.ObjectId
  shippingName    String
  shippingAddress String
  shippingCity    String
  shippingState   String
  shippingPincode String
  shippingPhone   String
  paymentMethod   PaymentMethod @default(COD)
  subtotal        Float
  totalDiscount   Float     @default(0)
  totalAmount     Float
  status          OrderStatus @default(PENDING)
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt
  user          User            @relation(fields: [userId], references: [id])
  items         OrderItem[]
  statusHistory StatusHistory[]
}
// ─── ORDER ITEM ─────────────────────────────────────────
model OrderItem {
  id               String  @id @default(auto()) @map("_id") @db.ObjectId
  orderId          String  @db.ObjectId
  productId        String  @db.ObjectId
  productName      String  // snapshot at time of order
  productImage     String  // snapshot at time of order
  quantity         Int
  unitPrice        Float   // original price at time of order
  discountedPrice  Float   // price after discount (= unitPrice if no discount)
  order    Order   @relation(fields: [orderId], references: [id])
  product  Product @relation(fields: [productId], references: [id])
}
// ─── WISHLIST ────────────────────────────────────────────
model Wishlist {
  id        String   @id @default(auto()) @map("_id") @db.ObjectId
  userId    String   @db.ObjectId
  productId String   @db.ObjectId
  createdAt DateTime @default(now())
  user    User    @relation(fields: [userId], references: [id])
  product Product @relation(fields: [productId], references: [id])
  @@unique([userId, productId]) // prevent duplicate wishlist entries
}
// ─── STATUS HISTORY ─────────────────────────────────────
model StatusHistory {
  id        String      @id @default(auto()) @map("_id") @db.ObjectId
  orderId   String      @db.ObjectId
  status    OrderStatus
  note      String?
  createdAt DateTime    @default(now())
  order Order @relation(fields: [orderId], references: [id])
}
// ─── ENUMS ──────────────────────────────────────────────
enum Role {
  CUSTOMER
  ADMIN
}
enum UserStatus {
  ACTIVE
  BLOCKED
}
enum Category {
  RINGS
  NECKLACES
  EARRINGS
  BRACELETS
  BANGLES
  OTHER
}
enum ProductStatus {
  ACTIVE
  INACTIVE
}
enum DiscountType {
  PERCENTAGE
  FLAT
}
enum OrderStatus {
  PENDING
  PROCESSING
  SHIPPED
  DELIVERED
  CANCELLED
}
enum PaymentMethod {
  COD
  CARD
  UPI
}

6. Discount Price Calculation
The backend must calculate discountedPrice before saving an OrderItem or returning a product to the frontend. Use this logic:
function calcDiscountedPrice(product) {
  const today = new Date();
  const isActive =
    product.discountEnabled &&
    product.discountStart <= today &&
    product.discountEnd   >= today;
  if (!isActive) return product.price;
  if (product.discountType === 'PERCENTAGE') {
    const saving = (product.price * product.discountValue) / 100;
    return Math.max(0, product.price - saving);
  }
  if (product.discountType === 'FLAT') {
    return Math.max(0, product.price - product.discountValue);
  }
  return product.price;
}

5. Authentication Flow
5.1 User Registration & Login
User submits email + password to POST /api/auth/register or POST /api/auth/login
Backend hashes password with bcryptjs on register, compares hash on login
On success, server signs a JWT: payload { userId, role: 'customer' }, signed with JWT_SECRET
Token returned in response body, stored in localStorage as 'karli_user_token'
All protected requests include header: Authorization: Bearer <token>
authMiddleware.js verifies and decodes the token, attaches user to req.user
5.2 Admin Login
Admin posts to POST /api/auth/admin-login (separate endpoint)
Backend checks role === 'admin' before issuing token
Admin JWT stored separately as 'karli_admin_token' in localStorage
adminMiddleware.js checks valid JWT AND req.user.role === 'admin'
5.3 Frontend Route Protection
ProtectedRoute.jsx wraps all customer-only pages — checks 'karli_user_token', redirects to /login if missing or expired
AdminRoute.jsx wraps all /admin/* pages — checks 'karli_admin_token', redirects to /admin/login if invalid

No Redux or external state library. All global state is handled with React Context API and the useState / useReducer hooks.
AuthContext
Stores logged-in user object, token, and login/logout functions. Reads from localStorage on app load to persist sessions across refreshes.
CartContext
Stores cart items array with add, remove, and update quantity functions. Persists to localStorage for all users.
WishlistContext
Stores wishlist product IDs. Fetched from the API on login and cleared on logout.

box-shadow: 0 0 16px rgba(212,175,55,0.45). Duration 200ms ease.

Full-screen overlay fades in (opacity). Nav links slide down with stagger delay (50ms each).
